What is Quizzical?

Quizzical is a React application that generates 5 randomly selected questions from the Open Trivia Database API for a user to test their trivia knowledge. The user can check their answers and then generate an endless set of quizzes.

Achievements

This project pushed me to not only synthesize my recently acquired skills in React hooks and functional components, but also to develop my ability to break problems down into smaller parts. This was a bigger project than I'm used to, culminating in more than a week's worth of trial and error.

Stumbling Blocks

There were two main roadblocks for me during this project: holding each individually selected answer choice in state, and displaying the number of correct answers at the end of the quiz.

For the first issue, I had to get around the different layers of data to reach the answers data using the map() method and event.target. At the second level, I isolated the current question and answer set by comparing the selected answer's parentNode.id to the id in the mapped array. Once I crossed that barrier, I looked for the answer choice that matched the selected choice's textContent to alter the appearance of the clicked choice.

Displaying the number of correct answers took longer to solve. I alternated from receiving the score from the Answers component or from the main App component. However, when I tried to save the state of the score in Answers.js, the returned state would run endlessly and crash the quiz. Ultimately, I settled on storing the selected answer as a key in the quiz's state, then compared the selected answer to the correct answer and incremented the score when the two keys matched up.

Future Enhancements

I believe there has to be a better way to organize the quiz state data so as to not use the selected and correct answers in multiple spots.

Further, after learning about React Router, I realized I could place my overlay and main content into two different routes rather than changing the display style of the overlay and working around the quirks of the colored blobs on the screen.

Finally, the OTBD API allows for the developer to change the number of questions returned, the category, and the difficulty. Implementing a component for the user to customize their own game would require altering the endpoints of the url to be more dynamic and altering the display of the questions within a single page or mutliple pages depending on the number of questions requested.
